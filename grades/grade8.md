# Программа по информатике: 8 класс
**Нагрузка:** 2 часа в неделю (34 блока / пары по 90 минут, всего 68 часов)

## Обзор программы
Курс 8 класса ориентирован на практическое программирование и создание веб-интерфейсов. В первом триместре ученики углубленно осваивают структурное программирование на языке Python, проходя путь от простейших математических вычислений до разработки собственной текстовой ролевой игры с ветвлением сюжета и инвентарем. Второй и третий триместры посвящены технологиям веб-разработки (дизайн и верстка HTML/CSS, основы клиентской интерактивности на JavaScript) и сетевой информационной безопасности.

## Распределение часов по триместрам
* **Триместр 1:** Структурное программирование на Python (12 пар, 24 часа)
* **Триместр 2:** Веб-технологии: разметка HTML и стилизация CSS (12 пар, 24 часа)
* **Триместр 3:** Клиентская веб-динамика (JS) и информационная безопасность (10 пар, 20 часов)

## Поурочное планирование (34 спаренных урока)

### Триместр 1. Структурное программирование на Python

#### Блок 1: Переменные, выражения и типы данных в Python
* **Теория:** Стандарты объявления переменных в Python (snake_case). Типы данных: целые (`int`), вещественные (`float`), строки (`str`), логический тип (`bool`). Операторы приведения типов.
* **Практика:** Создание интерактивной викторины в консоли с подсчетом дробного бала.
* **Проект:** Старт проекта «Текстовая RPG-игра»: проектирование характеристик игрока (сила, здоровье, золото).

#### Блок 2: Ветвления и логические операторы
* **Теория:** Условные операторы и блоки `if`, `elif`, `else`. Сложные условия с логическими связками `and`, `or`, `not`. Введение в PEP 8 (стандарты форматирования кода).
* **Практика:** Программа «Тестирование по кибербезопасности» с разветвленными вердиктами на основе набранных баллов.
* **Проект:** Создание сюжетного развилка в игре: выбор класса персонажа (Воин, Маг, Вор).

#### Блок 3: Итерации: Цикл `while`
* **Теория:** Цикл с предусловием `while`. Условия бесконечного выполнения циклов. Использование операторов управления циклом `break` и `continue`.
* **Практика:** Консольный экономический симулятор «Угадай число» с ограничением количества попыток и подсказками.
* **Проект:** Реализация главного игрового цикла (game loop) для текстовой игры.

#### Блок 4: Цикл `for` и последовательности
* **Теория:** Цикл прохода по диапазону `for i in range()`. Вложенные циклы.
* **Практика:** Отрисовка текстовых геометрических фигур (прямоугольники, пирамиды из звездочек `*`) в консоли.
* **Проект:** Создание механики тренировки: циклическое выполнение ударов по манекену с накоплением очков опыта.

#### Блок 5: Индексация и срезы строк
* **Теория:** Строки как неизменяемые последовательности символов. Положительные и отрицательные индексы. Срезы строк `[start:stop:step]`. Методы строк (`upper`, `lower`, `split`, `join`).
* **Практика:** Разработчик утилиты проверки паролей на содержание букв разного регистра и спецсимволов.
* **Проект:** Программа парсинга команд игрока (очистка ввода от пробелов, приведение в нижний регистр).

#### Блок 6: Списки (`list`) в Python. Базовые операции
* **Теория:** Списки как изменяемые упорядоченные контейнеры. Операции сложения списков, проверка вхождения `in`. Срезы списков.
* **Практика:** Создание динамического списка дел (ToDo list) с функциями добавления и просмотра задач.
* **Проект:** Создание системы инвентаря игрока (хранение предметов, оружия).

#### Блок 7: Методы списков и функции агрегации
* **Теория:** Методы добавления, удаления и поиска элементов (`append`, `insert`, `remove`, `pop`, `index`, `count`). Функции `len()`, `min()`, `max()`, `sum()`. Сортировка `sort()` и `sorted()`.
* **Практика:** Анализ логов сервера: нахождение пикового количества запросов за день с помощью методов списков.
* **Проект:** Изменение инвентаря игрока: сортировка предметов, лимит переносимого веса.

#### Блок 8: Словари (`dict`) как ассоциативные массивы
* **Теория:** Пары "ключ-значение". Ограничения на ключи словаря. Методы словарей (`keys`, `values`, `items`, `get`). Обработка отсутствующих ключей.
* **Практика:** Телефонный справочник с возможностью быстрого поиска, редактирования и удаления контактов.
* **Проект:** Описание комнат и локаций текстового мира в виде словаря словарей.

#### Блок 9: Создание функций и модульность
* **Теория:** Функции пользователя. Локальные и глобальные области видимости переменных. Аргументы по умолчанию. Инструкция `return`.
* **Практика:** Написание математического модуля с функциями расчета силы урона персонажа от разного типа брони.
* **Проект:** Выделение механики боя и механики перемещения в отдельные функции.

#### Блок 10: Текстовые файлы на Python. Углубленный Obsidian и Logseq
* **Теория:** Потоковая работа с файлами в Python: менеджер контекста `with open()`, чтение/запись, кодировка UTF-8. Концепция Outliner-редакторов (Logseq) против традиционных баз (Obsidian). Организация личной базы знаний (PKM): двусторонние ссылки (Backlinks), теги, фильтрация данных с помощью плагина Dataview.
* **Практика:** Написание на Python скрипта ведения дневника, экспортирующего записи в файлы Markdown. Импорт логов в Obsidian/Logseq, создание связей и автоматическая фильтрация Dataview по тегам/дате.
* **Проект:** Интеграция сохранения прогресса игры в структурированные Markdown-файлы для импорта в базу знаний.

#### Блок 11: Обработка исключений, отладка и ИИ-код-просмотрщик (1 час ИИ)
* **Теория:** Ошибки времени выполнения. Конструкция `try - except`. **Использование ИИ (1 час):** ИИ как рецензент кода. Как передавать код ИИ для выявления логических нестыковок, бесконечных циклов в игре и форматирования по PEP8.
* **Практика:** Написание отказоустойчивого калькулятора. Запрос ИИ-помощнику для ревью кода игры на предмет обнаружения утечек логики (например, некорректной обработки инвентаря).
* **Проект:** Внедрение отладочных блоков в игру и репутационный рефакторинг при помощи ИИ.

#### Блок 12: Защита проектов: Текстовая RPG-игра
* **Теория:** Оценка качества кода (Code review). Признаки чистого кода в Python.
* **Практика:** Играем в игры друг друга, ищем баги и логические лазейки.
* **Проект:** Защита комплексного проекта «Текстовая RPG-игра».

### Триместр 2. Веб-технологии: разметка HTML и стилизация CSS

#### Блок 13: Устройство веб-страниц. Язык разметки HTML5
* **Теория:** Как устроен веб? Что делают браузеры с HTML кодом. Структура HTML документа: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`. Кодировка.
* **Практика:** Написание первой веб-страницы в VS Code с заголовками, текстовыми абзацами и разделителями.
* **Проект:** Старт командного проекта «Сайт-портфолио класса или ученика». Создание макета в Markdown.

#### Блок 14: Базовые элементы HTML5: списки, ссылки, изображения
* **Теория:** Парные и непарные теги. Обязательные атрибуты тегов (для картинок `src`/`alt`, для ссылок `href`). Списки (`<ul>`, `<ol>`, `<li>`).
* **Практика:** Создание веб-каталога любимых полезных веб-ресурсов с описаниями и скриншотами.
* **Проект:** Разработка главной страницы классного сайта (информация об учениках, ссылки на профили).

#### Блок 15: Таблицы и формы в HTML5. Ввод пользовательских данных
* **Теория:** Структура таблиц (`<table>`, `<tr>`, `<td>`, `<th>`). Формы ввода (`<form>`), основные типы полей ввода (`<input type="...">`: text, email, password, button).
* **Практика:** Создание страницы регистрации пользователя с полями для ввода почты, пароля и анкетой.
* **Проект:** Разработка страницы обратной связи для классного сайта.

#### Блок 16: Мультимедиа элементы и блочные/строчные теги
* **Теория:** Внедрение видео (`<video>`) и аудио (`<audio>`). Понятие блочных (`<div>`, `<p>`, `<h1>`) и строчных (`<span>`, `<a>`, `<strong>`) тегов.
* **Практика:** Верстка страницы галереи классных видеороликов, смонтированных в 6 классе.
* **Проект:** Добавление мультимедиа-контента на сайт.

#### Блок 17: Знакомство со стилями CSS3
* **Теория:** Зачем отделять структуру от оформления. Способы подключения CSS (инлайновый, внутренний, внешний файл). Синтаксис CSS правил. Селекторы по тегам.
* **Практика:** Цветовая настройка и изменение шрифтов на созданной ранее HTML-странице через внешний CSS-файл.
* **Проект:** Создание файла стилей `style.css` и подключение его ко всем страницам сайта.

#### Блок 18: Селекторы классов (`.class`) и идентификаторов (`#id`)
* **Теория:** Как применить стили выборочно. Группировка правил. Псевдоклассы интерактивности дверей (`:hover`, `:active`).
* **Практика:** Оформление кнопок перехода по сайту с изменением цвета при наведении курсора.
* **Проект:** Разработка верхнего навигационного меню сайта (интерактивный Navbar).

#### Блок 19: Блочная модель CSS (Box Model)
* **Теория:** Свойства элемента: ширина/высота (`width`/`height`), внутренние отступы (`padding`), внешние поля (`margin`), рамка (`border`).
* **Практика:** Верстка карточек товаров/портфолио с закругленными рамками, тенью и отступами.
* **Проект:** Оформление персональных визиток учеников на главной странице сайта с помощью Box Model.

#### Блок 20: Макеты страниц: Свойства Flexbox в CSS
* **Теория:** Как выравнивать элементы на странице. Контейнер флексов `display: flex`. Направления осей (`flex-direction`). Выравнивание `justify-content` и `align-items`.
* **Практика:** Верстка шапки сайта и двухколоночного макета с использованием Flexbox.
* **Проект:** Разработка адаптивной сетки карточек портфолио с помощью Flexbox.

#### Блок 21: Веб-шрифты и позиционирование
* **Теория:** Подключение внешних шрифтов Google Fonts. Свойство `position` (static, relative, absolute, fixed).
* **Практика:** Создание «липкого» верхнего меню и всплывающей подсказки в нижнем углу экрана.
* **Проект:** Добавление красивых славянских шрифтов и позиционирование декораций сайта.

#### Блок 22: Адаптивная верстка и медиа-запросы
* **Теория:** Как сайты подстраиваются под мобильные устройства. Конструкция `@media`. Отслеживание ширины экрана.
* **Практика:** Написание CSS правил, которые скрывают тяжелую графику и перестраивают блоки в один столбец на узком экране (менее 600px).
* **Проект:** Адаптация портфолио класса под смартфоны.

#### Блок 23: Дебаг верстки в DevTools и ИИ как адаптивный макетчик (1 час ИИ)
* **Теория:** Использование DevTools браузера для инспекции CSS свойств. **Использование ИИ (1 час):** ИИ как помощник адаптивной верстки. Составление промптов для создания сложных правил CSS Media Queries и выравнивания блоков Flexbox на мобильных экранах.
* **Практика:** Исправление поломок макетов веб-страницы через инспектор стилей DevTools. Запрос к ИИ для генерации CSS-кода перестройки карточек портфолио в один столбец для экранов смартфонов.
* **Проект:** Верификация разметки классного сайта и адаптация сеток с помощью ИИ.

#### Блок 24: Выкатка сайта в локальную сеть

### Триместр 3. Клиентская веб-динамика (JS) и информационная безопасность

#### Блок 25: Плавные переходы и CSS-анимации
* **Теория:** Свойство плавности `transition` (задержки, функции сглаживания). Ключевые кадры `@keyframes` для сложных анимаций.
* **Практика:** Создание анимированной карточки, которая плавно переворачивается при наведении или летит по экрану.
* **Проект:** Добавление микро-анимаций в классный сайт.

#### Блок 26: Введение в язык программирования JavaScript (JS)
* **Теория:** Место JavaScript в стеке веб-технологий. Клиентский код. Базовый синтаксис: объявление переменных (`let`, `const`).
* **Практика:** Подключение JS-скрипта к HTML-странице через тег `<script>`. Вызов окна оповещения `alert()`.
* **Проект:** Создание интерактивной кнопки «Получить бонус в игре» на сайте.

#### Блок 27: Работа с элементами веб-страницы в JS. Объектная модель документа (DOM)
* **Теория:** Что такое DOM-дерево. Поиск элементов по ID (`document.getElementById`) или классу. Изменение текста и цвета элементов из JS.
* **Практика:** Написание простого переключателя тем оформления сайта (смена светлой темы на темную кликом по кнопке).

#### Блок 28: Обработка событий в JavaScript
* **Теория:** Слушатель событий `.addEventListener('click', function)`. Типы событий (клик, наведение, нажатие клавиши).
* **Практика:** Создание интерактивного счетчика кликов с анимацией расширения кнопки.
* **Проект:** Программирование переходов между вопросами квиза.

#### Блок 29: Веб-формы и расчеты на JS
* **Теория:** Чтение данных из текстовых полей ввода (`input.value`). Преобразование строк в числа.
* **Практика:** Создание калькулятора стоимости аренды роликов на основе выбора опций (флажки, переключатели).
* **Проект:** Реализация подсчета финального балла в интерактивном квизе непосредственно на странице.

#### Блок 30: Устройство безопасных веб-соединений. HTTPS и SSL/TLS
* **Теория:** Отличие HTTP от HTTPS. Симметричное и асимметричное шифрование. Роль сертификатов безопасности.
* **Практика:** Анализ сертификатов безопасности различных сайтов через замочек в строке браузера.

#### Блок 31: Уязвимости веб-ресурсов (SQL-инъекции, XSS)
* **Теория:** Внедрение вредоносного кода в форму. Межсайтовый скриптинг (XSS). Как злоумышленники крадут пароли через формы.
* **Практика:** Разбор демонстрационного стенда с уязвимой формой комментариев: запуск скрипта `alert` через текстовое поле.
* **Проект:** Внедрение защиты (экранирования символов) в инпуты веб-квиза.

#### Блок 32: Безопасность облачных хранилищ и аккаунтов (МФА)

* **Практика:** Настройка двухфакторной аутентификации на учебных почтовых ящиках.
* **Проект:** Разработка подсистемы безопасности для сайта проекта: правила защиты администрирования.

#### Блок 33: Оптимизация производительности сайта и ИИ как JS-аудитор (1 час ИИ)
* **Теория:** Скорость веб-страниц, сжатия. **Использование ИИ (1 час):** ИИ как аудитор кода JavaScript. Как попросить ИИ проверить JS-логику квиза на безопасность (защита от читерства со стороны пользователей) и оптимизировать объемы скриптов.
* **Практика:** Оптимизация ресурсов проекта. Диалог с ИИ по проверке JS-кода игры-квиза и рефакторинг обработки событий клика.
* **Проект:** Итоговая сборка папок оптимизированного и проверенного ИИ сайта.

#### Блок 34: Демонстрация веб-инструментов и интерактивных квизов (Защита)
