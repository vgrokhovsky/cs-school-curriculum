# Программа по информатике: 10 класс
**Нагрузка:** 2 часа в неделю (68 учебных часов в год, уроки по 1 академическому часу / 45 минут)

## Обзор программы
Курс 10 класса ориентирован на развитие системного вычислительного мышления, навыков командной разработки цифровых продуктов и подготовку к Единому государственному экзамену (ЕГЭ). Ученики знакомятся с архитектурой современных систем, реляционными СУБД (на примере SQLite), языком запросов SQL и продвинутой работой в Python (библиотека `sqlite3`, алгоритмы сложности, рекурсивные деревья решений и теория игр). Инженерная линия построена на практиках **совместной командной разработки** с использованием контроля версий Git/GitHub.

### Проекты по триместрам:
* **Триместр 1 (Уроки 1–24):** Разработка структуры (схемы) реляционной базы данных (СУБД SQLite) командного проекта (например, «Система инвентаризации школы», «Каталог библиотеки» или «Учет учебных курсов») с SQL-скриптами инициализации и конфигурацией командного репозитория Git (правила веток, коммитов, Kanban-планирование).
* **Триместр 2 (Уроки 25–48):** Готовое консольное Python-приложение для администрирования базы данных SQLite (интеграция модуля `sqlite3`, операции создания, чтения, изменения и удаления записей — CRUD), покрытое тестами проверки целостности.
* **Триместр 3 (Уроки 49–68):** Программа-симулятор теории игр (для решения задач 19-21 ЕГЭ на Python) со встроенной аналитической панелью (веб-страницей дашборда на CSS Grid) и подготовленным презентационным Pitch Deck пакетом для инвесторов / жюри.

## Сценарии организации работы на уроках

### Сценарий 1: Индивидуальная траектория («Каждый сам»)
* Каждый ученик создает свой собственный проект базы данных.
* Самостоятельно настраивает Git-репозиторий и ведет свою историю коммитов.
* Индивидуально пишет код интеграции SQLite с Python и пишет математический симулятор.

### Сценарий 2: Групповая траектория (команды из 3 человек)
Ученики распределяются по ролям, соответствующим современным ИТ-командам:
* **Роль А — Менеджер продукта / Разработчик БД (Product Manager & DB Architect):** Отвечает за проектирование схемы данных (таблицы, связи, ключи), написание SQL-кода, планирование задач команды на Kanban-доске, ведение технической документации в README.md.
* **Роль Б — Ведущий разработчик логики (Lead Backend Developer):** Пишет код логических скриптов Python, интегрирует БД через `sqlite3`, разрабатывает математическое ядро алгоритмов теории игр и рекурсивных переборов, проводит code review кода коллег.
* **Роль В — Инженер интерфейсов / Тестировщик (Frontend & QA Engineer):** Создает веб-страницу дашборда на HTML/CSS Grid, подготавливает тестовые наборы данных, пишет тесты верификации запросов СУБД, отвечает за слияние веток Git и релизы на GitHub.
