# Программа по информатике: 10 класс
**Нагрузка:** 2 часа в неделю (34 блока / пары по 90 минут, всего 68 часов)

## Обзор программы
Курс 10 класса является вводным этапом среднего общего образования и закладывает прочный фундамент профессиональных ИТ-навыков и подготовки к Единому государственному экзамену (ЕГЭ). Программа включает изучение архитектуры современных вычислительных систем, командную работу с использованием распределенной системы контроля версий Git/GitHub, проектирование реляционных баз данных и написание запросов SQL (на примере SQLite). Во втором и третьем триместрах углубленно изучается алгоритмика на Python и методы компьютерного моделирования математических задач ЕГЭ.

## Распределение часов по триместрам
* **Триместр 1:** Архитектура ИТ, СУБД и контроль версий Git (12 пар, 24 часа)
* **Триместр 2:** Алгоритмизация на Python и структуры данных (12 пар, 24 часа)
* **Триместр 3:** Моделирование процессов и Единый госэкзамен (10 пар, 20 часов)

## Поурочное планирование (34 спаренных урока)

### Триместр 1. Архитектура ИТ, СУБД и контроль версий Git

#### Блок 1: Системная архитектура. Как думает процессор
* **Теория:** Принцип фон Неймана. Процессор (ALU, CU, регистры). Машинный код и ассемблер (базовое понятие). Логические вентили (AND, OR, NOT, XOR). Сложение двоичных чисел на транзисторах (сумматор).
* **Практика:** Моделирование работы полусумматора в цифровом симуляторе схем.
* **Проект:** Старт командного проекта «Система инвентаризации школы» — распределение ролей в командах по 3 человека.

#### Блок 2: Маршрутизация в сетях и маски подсетей
* **Теория:** Проектирование сетей. Понятие адреса сети, широковещательного адреса. Маска подсети ($/24$, $/16$ и т.д.). Перевод масок в двоичный вид.
* **Практика:** Расчет диапазона доступных IP-адресов по выданному IP хоста и маске. Настройка локальной маршрутизации.
* **Проект:** Проектирование сетевой архитектуры базы данных проекта инвентаризации.

#### Блок 3: Контроль версий: Инициализация репозитория, коммиты и утечки секретов
* **Теория:** Роль VCS (систем контроля версий). Состояния файлов в Git. Информационная безопасность при разработке: предотвращение случайного коммита паролей, секретов и токенов доступа в репозиторий. Настройка и роль файла `.gitignore` для исключения системных настроек и личных конфигурационных файлов.
* **Практика:** Первоначальная настройка Git (`git config`), инициализация репозитория (`git init`), создание защитного `.gitignore`. Проведение индексации (`git add`) и коммитов (`git commit`), анализ логов.
* **Проект:** Инициализация локального репозитория проекта инвентаризации с защищенной структурой файлов.

#### Блок 4: Ветвление в Git: Работа в параллельных ветках
* **Теория:** Зачем нужны ветки (`branches`). Создание и переключение веток (`git checkout -b`). Слияние изменений (`git merge`). Понятие конфликта слияния (Merge Conflict) и способы его ручного разрешения.
* **Практика:** Имитация работы двух разработчиков над одним файлом, возникновение конфликта и его ручное исправление в коде.
* **Проект:** Создание веток под задачи участников проекта (БД, интерфейс, логика).

#### Блок 5: Облачные репозитории GitHub и авторизация (SSH vs HTTPS tokens)
* **Теория:** Облачные репозитории. Команды отправки (`git push`), стягивания (`git pull`) и клонирования (`git clone`). Безопасность авторизации на удаленных ИТ-сервисах: использование временных токенов (Personal Access Tokens) и криптографических ключей доступа SSH (открытый и закрытый ключи). Безопасный рабочий процесс совместной разработки: правила Pull Request и процедура взаимного код-ревью перед слиянием.
* **Практика:** Создание и привязка репозитория к GitHub. Настройка защищенной авторизации. Тестирование рабочего процесса в команде: отправка ветки, создание Pull Request, код-ревью.
* **Проект:** Связывание локального проекта с командным репозиторием GitHub через шифрованный канал.

#### Блок 6: Введение в базы данных. Реляционная модель
* **Теория:** Что такое СУБД. Реляционные базы данных. Ключевые понятия: сущность, связь, кортеж, атрибут. Типы связей (один-к-одному, один-ко-многим, многие-ко-многим).
* **Практика:** Создание концептуальной ER-диаграммы базы данных школы на бумаге или в Excalidraw.
* **Проект:** Разработка схемы таблиц базы данных инвентаризации.

#### Блок 7: Проектирование баз данных в SQLite. Типы данных
* **Теория:** Легковесная СУБД SQLite. Установка DB Browser for SQLite. Типы данных SQL (INTEGER, TEXT, REAL, BLOB, NULL). Первичные ключи (`PRIMARY KEY`), внешние ключи (`FOREIGN KEY`).
* **Практика:** Создание структуры БД проекта в DB Browser, ручное заполнение тестовыми данными.
* **Проект:** Генерация файла базы данных `inventory.db` при помощи визуального инструмента СУБД.

#### Блок 8: Написание SQL запросов: Выборка и фильтрация
* **Теория:** Декларативный синтаксис SQL. Базовая структура: `SELECT columns FROM table WHERE condition`. Операторы логического сравнения, `LIKE` с масками `%` и `_`, сортировка `ORDER BY`.
* **Практика:** Составление запросов к базе данных книг: вывод всех книг конкретного автора, изданных после 2010 года, по цене со скидкой.
* **Проект:** Создание поисковых SQL-запросов для инвентаря (поиск свободных парт, проекторов).

#### Блок 9: Агрегирующие функции и группировка в SQL
* **Теория:** Функции `COUNT()`, `SUM()`, `AVG()`, `MIN()`, `MAX()`. Группировка строк при помощи предложения `GROUP BY`. Фильтрация агрегированных данных через `HAVING`.
* **Практика:** Запросы к таблице продаж: подсчет общей выручки по каждому менеджеру за месяц, вывод только тех, чьи продажи превысили лимит.
* **Проект:** Разработка отчетов по количеству списываемого оборудования по категориям.

#### Блок 10: Связывание нескольких таблиц в SQL (`JOIN`)
* **Теория:** Соединение данных из разных таблиц. Разница между `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN` (базово). Объединение таблиц по первичному и внешнему ключам.
* **Практика:** Написание SQL-запроса, выводящего имя ученика и фамилию его классного руководителя из связанных таблиц учеников и классов.
* **Проект:** Реализация связей между оборудованием, кабинетами и ответственными учителями.

#### Блок 11: Изменение данных в SQL (`INSERT`, `UPDATE`, `DELETE`)
* **Теория:** Управление данными в реляционной БД. Безопасность операций изменения данных. Транзакции (понятие ACID).
* **Практика:** Написание SQL скриптов добавления нового оборудования, списания старого оборудования (удаления) и перемещения столов между классами (обновление).
* **Проект:** Подготовка SQL-файлов инициализации и первоначального заполнения бд.

#### Блок 12: Защита проектов: База данных инвентаризации школы
* **Теория:** Проектирование ИТ-решений. Правила презентации ПО.
* **Практика:** Команды демонстрируют работоспособность базы данных, решая три проверочных SQL-запроса от преподавателя на лету. Оценка слаженности работы в Git.
* **Проект:** Сдача проекта «База данных инвентаризации школы».

### Триместр 2. Алгоритмизация на Python и структуры данных

#### Блок 13: Рекурсивные алгоритмы на Python
* **Теория:** Что такое рекурсия. Разделяй и властвуй. Базовый случай рекурсии (условие выхода) и рекурсивный шаг. Опасность переполнения стека (Call Stack Overflow).
* **Практика:** Написание рекурсивных функций вычисления факториала, чисел Фибоначчи. Сравнение скорости работы рекурсии и обычного цикла.
* **Проект:** Старт проекта «Анализатор текстовых логов» на Python — создание рекурсивного обходчика вложенных папок с файлами логов.

#### Блок 14: Алгоритмы классической быстрой сортировки
* **Теория:** Понятие асимптотической сложности алгоритмов ($O$-нотация). Сортировка слиянием (Merge Sort). Быстрая сортировка Хоара (Quick Sort) как пример эффективного рекурсивного алгоритма.
* **Практика:** Реализация Quick Sort на Python. Замеры производительности сортировок $O(N^2)$ (пузырек) и $O(N \log N)$ (быстрая) на массивах из 100 000 случайных чисел.
* **Проект:** Разработка подсистемы сортировки логов по времени события.

#### Блок 15: Хэш-таблицы и Множества (`set`) в Python
* **Теория:** Быстрый поиск за постоянное время $O(1)$. Понятие хэш-функции. Встроенный тип данных `set`. Операции над множествами (объединение, пересечение, разность).
* **Практика:** Нахождение уникальных слов в тексте книги, определение общих слов у двух авторов с помощью пересечения множеств.
* **Проект:** Использование множеств для мгновенного отсечения дублирующихся строк логов.

#### Блок 16: Ассоциативные массивы (`dict`) для сложных группировок
* **Теория:** Оптимизация поиска ключей. Использование словарей для статистического подсчета. Вложенные словари.
* **Практика:** Написание скрипта, подсчитывающего частоту появления каждого символа (или слова) в заданном текстовом файле.
* **Проект:** Группировка лог-событий по кодам ошибок (например, подсчет количества ошибок 404 по дням).

#### Блок 17: Алгоритмы на графах. Поиск путей
* **Теория:** Способы представления графов в коде Python (список смежности в виде словаря списков). Алгоритмы обхода графа: Поиск в ширину (BFS) для нахождения кратчайшего пути, Поиск в глубину (DFS).
* **Практика:** Написание скрипта поиска кратчайшего пути между станциями метро.
* **Проект:** Моделирование сетевой топологии серверов, с которых собираются логи.

#### Блок 18: Обработка текстов. Регулярные выражения в Python
* **Теория:** Что такое регулярные выражения (Regular Expressions, RegEx). Синтаксис шаблонов. Python-модуль `re`. Поиск по шаблону (`re.search`, `re.findall`).
* **Практика:** Скрипт проверки валидности вводимых адресов электронной почты и номеров телефонов.
* **Проект:** Написание регулярного выражения для парсинга сложной строки лог-файла (выделение даты, IP-клиента, кода HTTP-ответа).

#### Блок 19: Чтение и парсинг файлов в форматах CSV и JSON
* **Теория:** Форматы обмена данными CSV (текст с разделителями) и JSON (текстовые объекты). Модули `csv` и `json` в Python. Сериализация и десериализация.
* **Практика:** Преобразование сырого списка словарей контактов в валидный JSON файл и экспорт в Excel-совместимый CSV.
* **Проект:** Сохранение структуры результатов анализа логов в формате JSON.

#### Блок 20: Объектно-ориентированное программирование (ООП): Классы и объекты
* **Теория:** Философия ООП. Класс как шаблон, объект как экземпляр. Атрибуты класса. Конструктор класса `__init__`. Методы объекта. Концепция инкапсуляции (приватные поля, геттеры и сеттеры).
* **Практика:** Разработка класса `Student` с оценками и расчетом среднего балла, защищенным от записи левых значений.
* **Проект:** Создание класса `LogEntry` для представления единичной записи лог-файла.

#### Блок 21: Наследование и полиморфизм в ООП
* **Теория:** Иерархия классов. Базовый и дочерний классы. Переопределение методов базового класса. Полиморфизм.
* **Практика:** Создание классов `AccessLogEntry` и `ErrorLogEntry`, наследующих общий класс `LogEntry` с переделкой логики форматирования вывода.
* **Проект:** Оптимизация парсера логов под разные классы сетевых событий.

#### Блок 22: Моделирование логических систем в ООП
* **Теория:** Композиция и агрегация объектов в ООП.
* **Практика:** Моделирование системы банкомата: класс `Bank`, класс `Account`, класс `Transaction`.
* **Проект:** Создание класса `LogAnalyzer`, управляющего коллекцией объектов `LogEntry` и фильтрующего их.

#### Блок 23: Автоматическое тестирование кода (Unit-тесты)
* **Теория:** Зачем нужно писать тесты? Пирамида тестирования. Знакомство с фреймворком `pytest` или встроенным `unittest`. Модульные тесты.
* **Практика:** Написание тестов для проверки регулярных выражений парсера логов.
* **Проект:** Покрытие ключевых функций анализатора логов тестами.

#### Блок 24: Защита проекта: Инструмент анализа текстовых логов
* **Теория:** Проектирование CLI-инструментов, обработка аргументов запуска.
* **Практика:** Запуск анализатора на пакете реальных логов объемом 10 МБ. Генерация JSON отчета о сбоях системы. Оценка качества и покрытия тестами.
* **Проект:** Финальная сдача проекта «Анализатор логов на Python».

### Триместр 3. Моделирование процессов и Единый госэкзамен

#### Блок 25: Знакомство со структурой ЕГЭ по информатике
* **Теория:** Задачи КЕГЭ (Компьютерного ЕГЭ). Спецификация контрольных измерительных материалов. Разделение времени.
* **Практика:** Первое знакомство с интерфейсом сдачи ЕГЭ на симуляторе. Решение 1–2 простейших задач.
* **Проект:** Старт курсового проекта исследования теории игр «Модель выигрышных стратегий».

#### Блок 26: Решение задач по системам счисления для ЕГЭ на Python
* **Теория:** Быстрый перевод чисел в любые системы счисления с помощью Python. Задачи ЕГЭ на нахождение сумм цифр выражения.
* **Практика:** Написание кода Python для вычисления выражений типа $3 \cdot 81^{12} - 9^4 + 27^{10}$ и перевода результата в 9-ричную систему.
* **Проект:** Написание модуля перевода чисел для теории игр.

#### Блок 27: Программное решение логических задач ЕГЭ (Задание 15)
* **Теория:** Алгоритмический перебор параметров логических уравнений. Нахождение промежутков на координатной оси или делимости.
* **Практика:** Написание программы перебора значений логических формул для нахождения максимального параметра $A$ в условии конъюнкций.
* **Проект:** Добавление системы поддержки логического анализа в проект.

#### Блок 28: Теория игр на Python: Модель игры Ним с 1 кучей
* **Теория:** Ход игры в Ним. Выигрышные и проигрышные стратегии. Задачи 19–21 ЕГЭ. Алгоритмическое дерево ходов.
* **Практика:** Написание рекурсивной функции на Python, определяющей возможные ходы и вычисляющей победителя за минимальное число ходов при игре с одной кучей.
* **Проект:** Разработка симулятора игры Ним с одной кучей с выводом всех возможных исходов.

#### Блок 29: Теория игр на Python: Модель с 2 кучами
* **Теория:** Математика игры Ним с 2 кучами. Древовидная рекурсия. Мемоизация (кэширование результатов вызова функций `lru_cache`).
* **Практика:** Программирование выигрышного решения Ним с 2 кучами. Определение позиций первого и второго игрока по баллам.
* **Проект:** Расширение симулятора игры до 2 куч с построением таблицы выигрышных стратегий.

#### Блок 30: Рекурсивные уравнения ЕГЭ (Задание 16)
* **Теория:** Выражения рекуррентных соотношений. Оптимизация глубины рекурсии Python (`sys.setrecursionlimit`).
* **Практика:** Решение примеров ЕГЭ вида $F(n) = 2 \cdot F(n-1) + n$ при больших значениях $n$.

#### Блок 31: Динамическое программирование (Задания 22-23 ЕГЭ)
* **Теория:** Метод динамического программирования. Подсчет путей исполнителя «Кузнечик» или «Калькулятор» с ограничением ходов.
* **Практика:** Кодирование поиска количества программ, которые преобразуют число $A$ в число $B$, обязательно проходя через $C$ и минуя $D$.

#### Блок 32: Обработка больших числовых последовательностей
* **Теория:** Работа с файлами чисел. Поиск пар чисел по условиям деления разности, остаткам. Оптимизация прохода списков от $O(N^2)$ к $O(N)$ по времени.

#### Блок 33: Оптимизационные игры на основе теории игр
* **Теория:** Анализ сложности расчетов теории игр.
