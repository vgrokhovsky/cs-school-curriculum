# Программа по информатике: 10 класс
**Нагрузка:** 2 часа в неделю (68 учебных часов в год, уроки по 1 академическому часу / 45 минут)

## Обзор программы
Курс 10 класса ориентирован на развитие системного вычислительного мышления, навыков командной разработки цифровых продуктов и подготовку к Единому государственному экзамену (ЕГЭ). Ученики знакомятся с архитектурой современных систем, реляционными СУБД (на примере SQLite), языком запросов SQL и продвинутой работой в Python (библиотека `sqlite3`, алгоритмы сложности, рекурсивные деревья решений и теория игр). Инженерная линия построена на практиках **совместной командной разработки** с использованием контроля версий Git/GitHub.

### Проекты по триместрам:
* **Триместр 1:** Разработка структуры (схемы) реляционной базы данных (СУБД SQLite) командного проекта (например, «Система инвентаризации школы», «Каталог библиотеки» или «Учет учебных курсов») с SQL-скриптами инициализации и конфигурацией командного репозитория Git (правила веток, коммитов, Kanban-планирование).
* **Триместр 2:** Готовое консольное Python-приложение для администрирования базы данных SQLite (интеграция модуля `sqlite3`, операции создания, чтения, изменения и удаления записей — CRUD), покрытое тестами проверки целостности.
* **Триместр 3:** Программа-симулятор теории игр (для решения задач 19-21 ЕГЭ на Python) со встроенной аналитической панелью (веб-страницей дашборда на CSS Grid) и подготовленным презентационным Pitch Deck пакетом для инвесторов / жюри.

## Сценарии организации работы

### Сценарий 1: Индивидуальная траектория («Каждый сам»)
* Каждый ученик создает свой собственный проект базы данных.
* Самостоятельно настраивает Git-репозиторий и ведет свою историю коммитов.
* Индивидуально пишет код интеграции SQLite с Python и пишет математический симулятор.
* **Результат:** Индивидуальное СУБД-приложение с дашбордом.

### Сценарий 2: Групповая траектория (команды по 3 человека)
Ученики распределяются по ролям, соответствующим современным ИТ-командам:
* **Роль А — Менеджер продукта / Разработчик БД (Product Manager & DB Architect):**
  * Отвечает за проектирование схемы данных (таблицы, связи, ключи), написание SQL-кода, планирование задач команды на Kanban-доске, ведение технической документации в README.md.
* **Роль Б — Ведущий разработчик логики (Lead Backend Developer):**
  * Пишет код логических скриптов Python, интегрирует БД через `sqlite3`, разрабатывает математическое ядро алгоритмов теории игр и рекурсивных переборов, проводит code review кода коллег.
* **Роль В — Инженер интерфейсов / Тестировщик (Frontend & QA Engineer):**
  * Создает адаптивный веб-дашборд на HTML/CSS Grid для вывода аналитики, разрабатывает тестовые наборы данных, пишет тесты верификации запросов СУБД, отвечает за слияние веток Git и релизы на GitHub.

### Триместр 2. Разработка СУБД-приложения на Python (CRUD) (24 часа)

#### Урок 25: Интеграция Python с базами данных SQLite
* **Теория:** Подключение библиотек. Встроенный модуль `sqlite3`. Соединение с базой данных `sqlite3.connect()`. Понятие курсора (`cursor`).
* **Практика:** Написание Python-кода, открывающего файл базы `inventory.db` и выводящего версию SQLite.
* **Сам/Группа:** Старт проекта 2 триместра: Консольный менеджер СУБД на Python.

#### Урок 26: Выполнение SQL-запросов из Python
* **Теория:** Метод `cursor.execute()`. Извлечение результатов выборки: метод `fetchall()` (получить список всех записей) и `fetchone()`.
* **Практика:** Python-программа считывает все записи таблицы оборудования и выводит их в консоль.
* **Сам/Группа:** Настройка форматирования строк консольного вывода.

#### Урок 27: Параметризованные SQL-запросы в Python. Защита от SQL-инъекций
* **Теория:** Информационная безопасность СУБД. Опасность инъекций SQL при сложении строк. Защита с помощью подстановки параметров через кортежи `(?, ?)`.
* **Практика:** Разработка формы безопасного поиска техники по названию, вводимому пользователем.
* **Сам/Группа:** Исправление небезопасных конкатенаций строк в коде Python.

#### Урок 28: Изменение данных в БД через Python. Commit транзакций
* **Теория:** Запись и изменение данных в Python. Метод соединения `connection.commit()`. Логика отката транзакций `rollback()` при ошибках.
* **Практика:** Программа на Python добавляет новую запись в БД, подтверждая изменения.
* **Сам/Группа:** Внедрение метода логического коммита.

#### Урок 29: Разработка консольного CRUD интерфейса: Read (Чтение)
* **Теория:** Принцип REST/CRUD. Написание пользовательских функций чтения записей.
* **Практика:** Разработка функции `show_all_items()`, выводящей красивую текстовую таблицу остатков оборудования.
* **Сам/Группа:** Настройка обработки пустых таблиц.

#### Урок 30: Разработка CRUD интерфейса: Create (Создание)
* **Теория:** Запрос параметров ввода у пользователя, валидация типов.
* **Практика:** Создание функции `add_new_item()`, запрашивающей текстовое имя, количество и цену, и записывающей в БД.
* **Сам/Группа:** Обработка ошибок ввода пользователя (буквы вместо цены) в блоке `try - except`.

#### Урок 31: Разработка CRUD интерфейса: Update (Изменение)
* **Теория:** Логика изменения данных. Предварительная проверка существования изменяемой записи по ID.
* **Практика:** Функция `update_item_qty()`, изменяющая количество оборудования в конкретном кабинете по ID.
* **Сам/Группа:** Реализация пошагового интерфейса изменения данных.

#### Урок 32: Разработка CRUD интерфейса: Delete (Удаление)
* **Теория:** Безопасное удаление. Запросы подтверждения удаления («Вы уверены? Y/N»).
* **Практика:** Реализация функции `delete_item()`, удаляющей запись по ID после подтверждения пользователя.
* **Сам/Группа:** Внедрение логических ограничений удаления связанных объектов.

#### Урок 33: Объединение CRUD в единое консольное меню приложения
* **Теория:** Архитектура CLI-приложения. Цикл управления.
* **Практика:** Сборка функций CRUD в единый цикл обработки команд: «1 — Показать всё, 2 — Добавить, 3 — Изменить, 4 — Удалить, 5 — Выйти».

* **Теория:** Обработка системных сбоев СУБД в коде (например, нарушение уникальности ключа `IntegrityError`).

* **Сам/Группа:** Обеспечение стабильности кода консольного CLI-менеджера.

#### Урок 35: Автоматизированное тестирование функций СУБД
* **Теория:** Написание тестов проверки корректности записи данных. Имитация базы данных в оперативной памяти с помощью `:memory:`.
* **Практика:** Написание тестового скрипта на Python, разворачивающего временную базу и проверяющего работу функции добавления записи.
* **Сам/Группа:** Запуск локального тестирования кода.

#### Урок 36: Алгоритмы: Хэш-таблицы в решении задач ЕГЭ
* **Теория:** В вычислительном мышлении: понятие хэширования, хэш-таблиц. Быстрый поиск за время $O(1)$. Решение задач ЕГЭ на анализ числовых множеств (на примере отбора уникальных пар).
* **Практика:** Реализация быстрого поиска дубликатов кодом Python.
